Open source projects in research administration need more than a code repository. They
need clear stewardship, visible decision-making, and practical contribution pathways.

This resource collection should gather governance models, examples, and principles that
help institutions understand how community ownership works in practice.

For AI4RA, governance should not be treated as an afterthought once the technical work is
finished. It is part of what makes the technical work credible in the first place,
especially for tools that touch institutional operations, data interpretation, or AI-enabled
decision support.
